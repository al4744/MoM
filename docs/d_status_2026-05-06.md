## What runs on the VM (in order)

```bash
# Activate, sanity-check
cd ~/MoM
source .venv/bin/activate
PYTHONPATH=. python scripts/check_ablation_configs.py     # all 8 should be ✓

# After current matrix finishes, in tmux:
TS=$(ls -d results/d-run-* | tail -1 | sed 's|results/d-run-||')

# Run the new isolation + ablation configs (~5 configs × 3 min each ≈ 15 min)
for cfg in prefix_cache_only retention_no_ema retention_no_per_tool retention_no_pin; do
    PYTHONPATH=. python evaluation/run_eval.py \
        --config configs/${cfg}.yaml \
        --output results/d-run-$TS/${cfg}/ \
        2>&1 | tee results/d-run-$TS/${cfg}.log
done

# Generate cross-config + per-delta tables
PYTHONPATH=. python evaluation/comparison_table.py \
    --results-root results/d-run-$TS/ \
    --output results/d-run-$TS/comparison.md

for cfg in prefix_cache_only retention retention_int8 retention_int4 \
           retention_no_ema retention_no_per_tool retention_no_pin; do
    PYTHONPATH=. python evaluation/comparison_table.py \
        --ablate results/d-run-$TS/baseline/summary.json \
                 results/d-run-$TS/${cfg}/summary.json \
        --output results/d-run-$TS/ablate-baseline-vs-${cfg}.md
done

# THE HEADLINE: prefix_cache_only → retention isolates pin-or-evict
PYTHONPATH=. python evaluation/comparison_table.py \
    --ablate results/d-run-$TS/prefix_cache_only/summary.json \
             results/d-run-$TS/retention/summary.json \
    --output results/d-run-$TS/ablate-pco-vs-retention.md

# Verify retention fired
for cfg in retention retention_int8 retention_int4 retention_no_ema \
           retention_no_per_tool retention_no_pin; do
    f=results/d-run-$TS/$cfg/events.jsonl
    [ -f "$f" ] || continue
    echo "$cfg: pins=$(grep -c '"event_type":"pin"' $f) reuses=$(grep -c '"event_type":"reuse"' $f) reject=$(grep -c '"event_type":"pin_rejected_budget"' $f)"
done
```

## Accuracy benchmark (lm-eval-harness, MMLU)

`scripts/run_accuracy_eval.py` reuses our `build_retention_config` so the
accuracy numbers are produced by the same engine as the latency numbers,
cell-for-cell.

```bash
pip install "lm_eval[vllm]"

# One config, default MMLU, time-bounded by --limit
PYTHONPATH=. python scripts/run_accuracy_eval.py \
    --config configs/retention.yaml \
    --output results/d-run-$TS/retention/ \
    --tasks mmlu --limit 50

# Sweep across configs
for cfg in baseline prefix_cache_only retention retention_int8 retention_int4; do
    PYTHONPATH=. python scripts/run_accuracy_eval.py \
        --config configs/${cfg}.yaml \
        --output results/d-run-$TS/${cfg}/ \
        --tasks mmlu --limit 50
done
```

Output lands as `accuracy.json` per config and updates `summary.json` with
`mean_task_accuracy`, automatically picked up by the comparison table.

Time budget on L4:
| `--limit` per subtask | Total time, 5 configs |
|---|---|
| 20 | ≈ 15 min |
| 50 | ≈ 40 min |
| 200 | ≈ 2.5 h |
| no limit (full MMLU) | ≈ 7.5 h |

Recommend `--limit 50` for the v1 paper, full MMLU as appendix only if time.
